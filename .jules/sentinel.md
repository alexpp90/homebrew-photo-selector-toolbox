## 2024-07-01 - DNS Rebinding SSRF Vulnerability in Ollama Config
**Vulnerability:** A simple string-based check on a hostname (e.g., `hostname == "169.254.169.254"`) is insufficient for preventing SSRF attacks to cloud metadata endpoints because it can be bypassed via alternative IP representations (e.g., hex `0xa9fea9fe`) or DNS resolution (a custom domain pointing to `169.254.169.254`).
**Learning:** `urllib.parse.urlparse().hostname` extracts the host string, which requires resolution via `socket.gethostbyname` and IP parsing via `ipaddress` for proper link-local address filtering. However, note that a Time-of-Check to Time-of-Use (TOCTOU) vulnerability still theoretically exists because `urllib` performs a separate internal DNS resolution upon request execution.
**Prevention:** Use robust IP parsing instead of string matching. For complete protection against DNS rebinding, the HTTP client must be pinned to the exact IP resolved during the check phase, or a custom socket factory must be used to block link-local IPs at the socket level.
