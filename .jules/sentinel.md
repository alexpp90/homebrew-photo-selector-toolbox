## 2024-05-18 - Prevent SSRF Bypasses via HTTP Redirects
**Vulnerability:** Even when initial hostname resolution correctly blocks cloud metadata IPs (e.g., `169.254.169.254`), `urllib.request.urlopen` follows HTTP 302 redirects by default. Attackers can supply a safe URL that redirects to the restricted IP, bypassing the protection entirely.
**Learning:** Initial IP validation is insufficient if the underlying HTTP client freely follows redirects to arbitrary destinations.
**Prevention:** Implement a custom `urllib.request.HTTPRedirectHandler` that overrides `redirect_request` to unconditionally raise a `URLError`, and inject it into the request chain using `urllib.request.build_opener(NoRedirectHandler)`. Remember to update test suite mocks from `urllib.request.urlopen` to `urllib.request.OpenerDirector.open` to match the new architecture.
