## 2025-02-28 - Secure SQLite WAL and Directory Permissions
**Vulnerability:** SQLite databases using WAL mode (`journal_mode=WAL`) create `-wal` and `-shm` files that were left with default `umask` permissions (e.g., `644`), exposing sensitive cached data to other users on the system, even if the primary `.db` file was securely `chmod`'d to `600`. The parent configuration directory also lacked restricted permissions upon creation.
**Learning:** `chmod`ing a primary SQLite file is insufficient when WAL mode is active, as auxiliary files are dynamically created with system defaults. Relying purely on file permissions without restricting the parent directory opens up potential data leakage and file replacement risks in multi-user environments.
**Prevention:** Secure the parent configuration directory to `700` (`stat.S_IRWXU`) immediately upon creation to prevent access to any current or future files within it. Additionally, proactively apply secure `600` permissions to the `-wal` and `-shm` paths whenever the database is initialized.
