## 2025-02-28 - Secure Directory and SQLite Temporary File Permissions
**Vulnerability:** The application's configuration and cache directories, as well as SQLite's temporary `-wal` and `-shm` sidecar files, were being created with default system permissions, potentially allowing unauthorized local users to access sensitive database contents.
**Learning:** SQLite in WAL (Write-Ahead Logging) mode creates temporary `-wal` and `-shm` files that can persist and contain sensitive data from recent transactions. Securing only the main `.db` file is insufficient if these sidecar files are left with default permissions.
**Prevention:** When setting up sensitive configuration directories, use `os.chmod` with `stat.S_IRWXU` (700) immediately upon creation. When securing SQLite databases, actively check for and secure the `-wal` and `-shm` files if they exist using `os.chmod` with `stat.S_IRUSR | stat.S_IWUSR` (600). Additionally, ensure any temporary files created during testing are cleaned up to avoid committing binary artifacts.
