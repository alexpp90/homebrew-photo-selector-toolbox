## 2024-05-18 - Fix N+1 SQLite query problem in multiprocessing
**Learning:** When using `ProcessPoolExecutor` with SQLite databases, querying or writing to the database from within worker functions creates connection overhead and locking contention for each individual task, exacerbating N+1 performance issues.
**Action:** Always pre-fetch database records in bulk from the main thread before submitting tasks, pass the pre-fetched data to the workers, and have the workers return any new calculations. The main thread should then batch-write updates back to the database in chunks to maintain performance and avoid memory constraints.
